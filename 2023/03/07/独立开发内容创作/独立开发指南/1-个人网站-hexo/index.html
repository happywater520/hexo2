本博客如何搭建一（hexo+github搭建）

发表于2023-03-07|更新于2023-10-09|与信息时代个体书独立开发指南
|阅读量:

github

创建一个叫 happywater520.github.io 的仓库 https://github.com/happywater520/happywater520.github.io.git 把仓库拉到本地

hexo

安装hexo npm install -g hexo-cli

在本地仓库新建一个空文件夹放hexo代码。右键打开 Git Bash Here，输入命令:hexo init初始化

输入 hexo g 和 hexo s，看到如下提示，打开浏览器访问 http://localhost:4000 可成功访问

hexo+github

修改hexo 目录下的 _config.yml 文件，在文件末尾添加你的仓库地址。

安装 hexo-deployer-git 插件 npm install hexo-deployer-git --save

输入命令 hexo d 将本地 blog 推送到 github 远程仓库，也可能需要输入 username & pwd，推送成功后，可访问 https://happywater520.github.io/

hexo+github+域名

解析域名，需要关掉加速器 ping happywater520.github.io（可以加上参数-4）

在你已购买的域名后点击解析添加两条解析记录。

创建 CNAME.txt 里面放域名 happywater520.top，再去掉txt。上传github看是否关联成功（也可以尝试直接添加）

访问 https://happywater520.github.io/ 就会跳转到你的域名地址 happywater520.top

（顺利的话半小时即可完成，全程自己的github操作，对方只需要买域名并给它添加两条解析）

1.在自己github账号，创建一个后缀为github.io新的仓库（比如xyiy.github.io），添加分支 gh-pages！（很重要）

2.对方申请好域名，并给域名添加如下两条解析：一条数值来自于仓库名字（比如xyiy.github.io），一条数值来自于 ping xyiy.github.io（关掉加速器，ping你的仓库名字 比如结果为185.199.108.153）

3.在source目录里面添加一个CNAME.txt 的文件，放入你的域名 （比如xyiy.top），再把.txt去掉。刷新仓库setting设置里page页，已添加自定义域名（如果没有CNAME，这是在page页添加自定义域名的话，刷新后又需要重新添加）

4.将仓库拉到本地 git clone https://github.com/happywater520/xyiy.github.io.git (写你的仓库地址)

5.进入仓库 带有 .git的目录 新建空文件夹 （比如叫 hexo），在里面右键打开 git bash 输入 hexo init 就会生成目录。

6.配置里面的 _config.yml ,添加你的仓库地址和分支。

7.在git bash 从main分支里面切换到gh-pages git checkout gh-pages ，安装 hexo-deployer-git 插件 npm install hexo-deployer-git --save，然后输入 hexo g ， hexo d，就已经把代码推送到仓库里面了

8.不出意外，可以通过域名访问啦，如果想测试本地访问，就 hexo clean ,hexo g , hexo s，然后网页访问 http://localhost:4000/ 。
